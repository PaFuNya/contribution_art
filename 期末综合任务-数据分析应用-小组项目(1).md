# 《数据分析应用》期末综合任务

任务形式：小组项目  
建议组队人数：3-5人  
完成周期：1-2周  
提交形式：项目报告 + 代码 + 讲解视频  

---

## 一、任务主题

请各小组围绕一个**真实数据分析问题**，独立完成从“数据获取”到“分析建模”再到“可视化表达与结论输出”的完整数据分析项目。

项目应尽量贴近真实应用场景，例如：

1. 电商用户行为分析与销量预测
2. 电影/音乐/游戏评分与推荐分析
3. 城市空气质量或天气变化趋势分析
4. 教育数据分析与学习表现预测
5. 金融/股票/基金历史数据分析
6. 餐饮、旅游、酒店评论情感分析
7. 社交媒体文本情感与热点主题分析
8. 企业经营、客户分群与用户画像分析

各组也可以自拟题目，但必须与本课程“数据分析应用”内容紧密相关。

---

## 二、项目要求

每个小组的项目必须至少包含以下内容：

### 1. 数据集准备

1. 数据集可**自行准备**。
2. 数据应尽量来自公开平台或真实业务场景，禁止直接使用课堂完全相同的数据。
3. 数据量应达到基本分析要求，建议：
   - 结构化表格数据不少于 `1000` 行；
   - 若为文本分析任务，文本样本不少于 `500` 条；
   - 若为时间序列任务，时间点不少于 `100` 个。
4. 必须在报告中说明数据来源、字段含义、采集方式和使用范围。

### 2. 数据分析流程

项目必须完整体现以下流程：

1. 研究问题定义
2. 数据采集或下载
3. 数据清洗与预处理
4. 探索性数据分析（EDA）
5. 可视化分析
6. 特征工程
7. 建模与结果评估
8. 结论总结与建议

### 3. 建模要求

每组项目至少选择以下方向之一：

1. 分类任务
   - 如用户流失预测、贷款审批预测、评论情感分类等
2. 回归任务
   - 如销量预测、价格预测、评分预测等
3. 聚类任务
   - 如客户分群、用户画像、消费群体识别等
4. 降维任务
   - 如 PCA 可视化、特征压缩等
5. 时间序列任务
   - 如销售额、气温、访问量趋势分析与预测
6. 文本分析任务
   - 如评论情感分析、关键词分析、主题分析等

要求至少使用 **1 种机器学习方法**。如果项目较强，鼓励进行多模型对比。

### 4. 大模型辅助分析要求

每组必须至少包含 **1 个“大模型辅助数据分析”环节**，例如：

1. 使用 API 调用大模型，对分析结果生成阶段性总结
2. 利用大模型自动生成数据分析报告初稿
3. 使用大模型对评论文本做结构化抽取
4. 使用 LangChain 组织分析流程
5. 让大模型输出 JSON 格式的分析结论

注意：

1. 大模型是辅助工具，不可完全替代数据分析过程。
2. 报告中必须明确说明：哪些环节由大模型辅助完成，哪些结论由小组自行验证。

## 三、数据集获取参考

1. [Kaggle](https://www.kaggle.com/)
   - 推荐优先选择带有明确字段说明、任务背景清晰的数据集
2. [UCI Machine Learning Repository](https://archive.ics.uci.edu/)
3. [阿里云天池](https://tianchi.aliyun.com/)
4. [GitHub 开源数据集仓库](https://github.com/)
5. 政府开放数据平台
   - 如国家统计局、各地政务开放平台、气象数据平台等
6. 豆瓣、微博、B站、招聘网站等公开页面数据
   - 可在合规前提下自行采集或使用别人整理好的公开数据

---

## 四、项目结构

建议项目最终至少包含以下文件或内容：

1. `README.md`
   - 项目简介、分工说明、运行方式
2. `data/`
   - 原始数据或数据下载说明
3. `notebook.ipynb` 或 `main.py`
   - 核心分析代码
4. `report.docx` 或 `report.pdf`
   - 项目报告
5. `slides.pptx`
   - 课堂汇报PPT
6. `video.mp4` 
   - 讲解视频

---

## 五、报告内容要求

项目报告建议包含以下部分：

1. 项目题目
2. 小组成员与分工
3. 研究背景与问题定义
4. 数据来源与字段说明
5. 数据清洗与预处理过程
6. EDA 与可视化分析
7. 特征工程过程
8. 建模方法与参数说明
9. 模型评估结果
10. 大模型辅助分析设计与结果
11. 项目结论与建议
12. 项目不足与可改进方向

---

## 六、讲解视频要求

每组提交 1 个项目讲解视频，建议时长控制在 `3-5` 分钟。

视频中建议重点展示：

1. 你们为什么选这个题目
2. 数据从哪里来
3. 做了哪些清洗和分析
4. 用了什么模型，效果如何
5. 大模型在项目中起到了什么作用
6. 最终结论与实际意义是什么

不建议只展示大段代码，应重点展示项目逻辑、关键图表和结论。

建议采用以下任一种形式：

1. PPT 讲解录屏
2. Jupyter Notebook 演示录屏
3. 代码运行过程加口头讲解

视频要求：

1. 画面清晰，能看清标题、图表和关键结果
2. 讲解简洁，结构完整
3. 时长控制在 3-5 分钟内
4. 文件命名规范，例如：XX组-数分期末综合任务讲解视频.mp4`

---

## 七、注意事项

1. 严禁直接照搬网络现成完整项目作为本组成果。
2. 可以参考网络资料和借助 AI 工具，但必须经过本组消化、验证和重构。
3. 代码运行失败、图表无法解释、结论与数据不一致，将明显影响成绩。
4. 每位成员都应参与实际工作。
